# Page Structure: 12 Sections

## Section Flow

```
1. Hero              -- Hook + self-select (developer/company)
2. Pain: Companies   -- "Hiring is broken" with hard numbers
3. Pain: Developers  -- "The vicious circle" with empathy
4. Solution          -- "Not a job board. Not a bootcamp."
5. How It Works: Co  -- 3 steps for companies
6. How It Works: Dev -- 3 steps for developers
7. Outcome Criteria  -- Core feature showcase
8. Challenge Seekers -- The "fun" angle
9. Numbers           -- Market statistics (impact/trust)
10. Comparison Table -- DevStage vs alternatives
11. FAQ              -- Objection handling
12. Final CTA        -- Two-path conversion
```

## Section 3: Pain -- Developers

## Section 4: Solution

## Section 5: How It Works -- Companies

**Background**: White
**Headline**: "Three Steps to Better Hiring"

**Steps (numbered cards)**:
1. "Post a Listing" / "Define what you need. Set measurable outcome criteria
   (at least 3). Choose duration, tech stack, and format."
2. "Review Applications" / "Motivated developers apply with a message.
   Accept the ones who fit."
3. "Evaluate Real Work" / "Review their actual output against your criteria.
   Rate each one: pass, partial, or fail. Hire with confidence."

**CTA**: "Post a Listing" -> `/register`
