# DevStage: Unique Positioning

## The Unoccupied Quadrant

No existing platform combines all three of DevStage's core features:

1. **Bidirectional listings** -- companies AND developers can post
2. **Real project work** -- not simulated tests or interviews
3. **Structured outcome reviews** -- pass/partial/fail per measurable criterion

### Closest Competitors Cover Only One Dimension

| Platform | Real Work | Bidirectional | Structured Outcomes |
|----------|-----------|---------------|---------------------|
| Automattic (internal) | Yes | No | No |
| Hired | No | Yes (dev profiles) | No |
| HackerRank Projects | Simulated | No | Partial |
| Toptal | Contract work | No | No (star ratings) |
| **DevStage** | **Yes** | **Yes** | **Yes** |

## Positioning Statement

> **"Work speaks louder than resumes."**

This differentiates from:
- **Exclusivity plays** (Toptal's "top 3%", Turing's "top 1%")
- **Cost plays** (Arc's "save 80%")
- **Assessment plays** (HackerRank's "skills > pedigree")

DevStage centers on **paid real-world work as the verification mechanism**.

## Feature-Level Differentiation

### Outcome Criteria (Core Innovation)
- Company listings **require minimum 3 measurable criteria**
- Examples: "Deploy to production", "Write 5+ PRs", "Test coverage >80%"
- Each criterion is reviewed as pass/partial/fail
- Overall recommendation: ready_to_hire / needs_practice / not_recommended
- Developer can consent to make review public + add response

**No competitor requires pre-defined, measurable success criteria.**

### Developer-as-Poster
- Developers can post listings seeking mentorship or collaboration
- Companies apply TO developer listings (power inversion)
- Breaks the traditional "company holds all power" dynamic

**No competitor lets developers post listings for companies to apply to.**

### Cross-Role Applications
- Developers apply to company listings
- Companies apply to developer listings
- Both sides actively participate in matching

### Visibility Controls
- Public (anyone can see) / Authenticated (logged-in users) / Private (author only)
- Gives control over listing exposure

## Competitive Moat

### Short-term
- First-mover in the bidirectional work-trial marketplace
- Outcome criteria system creates structured data no competitor has
- Network effects: more reviews = more trust = more participants

### Long-term
- Outcome review history becomes a **portable verified credential**
- Aggregate data reveals which criteria predict success
- Platform becomes the **standard for work-based skill verification**
